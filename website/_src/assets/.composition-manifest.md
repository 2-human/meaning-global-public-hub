- **Labels global:** `window.MG_REVIEW_LABELS` (optional; overrides `MG_REVIEW_CONFIG.REVIEW_LABELS`). Defaults are English ("Comments", "Add a comment", …).
- **Storage:** localStorage, keyed per page path (`mg_review_comments:{slug}`); reviewer name in `mg_review_reviewer`. No network. In-memory fallback if localStorage throws.
- **Lifecycle mode:** full (Active/Applied/Archived; Apply / Edit / Archive / Restore / Delete per state).
- **Commentable surface:** allowlist — `h1–h6, p, li, blockquote` ∪ `ANCHOR_TAGS_EXTRA` ∪ `[data-comment-target]`. Site chrome (`nav`, `header[role="banner"]`, `footer`, `[data-review-skip]`) skipped.

## Deviations from canonical (light-cut profile)
1. **localStorage-only persistence.** No Firebase, no auth — the private review link is the soft gate. The 4-op adapter interface (`subscribe`/`create`/`update`/`remove`) matches `firebase-rtdb-adapter.md` §"Adapter contract", so a Firebase adapter drops in at the marked §FIREBASE SEAM without touching call sites. Per-page storage key rather than a broad bucket + client filter.
2. **Name-prompt identity.** First comment prompts for a reviewer name (stored locally); no accounts.
3. **Flat row view.** Rows filter directly by status (Active→pending, Applied→applied, Archived→archived) rather than the grouped `groupStatus()` board; the three-state data model and transition events are unchanged.
